## Prettier

What is prettier?

Prettier enforces a consistent code style (i.e. code formatting that wonâ€™t affect
the AST) across your entire codebase because it disregards the original styling\*
by parsing it away and re-printing the parsed AST with its own rules that take the
maximum line length into account, wrapping code when necessary.

Why are we using it?

Prettier enforces standards from the development team of React, [Facebook](https://facebook.github.io/jsx/).
This allows for the codebase to be more uniformity and legible under one set standard. The beauty of having an unopinionated
language is the ability to decide how you want your code. However, when you start to work with large teams you need a set standard.

## Setup for our styleguide

### 1. Install ESLint & Prettier extensions for VSCode

Optional - Set format on save and any global prettier options

### 2. Install Packages

```
npm i -D eslint prettier eslint-plugin-prettier eslint-config-prettier eslint-plugin-node eslint-config-node
```

```
npx install-peerdeps --dev eslint-config-airbnb
```

### 3. A .prettierrc for prettier rules (semicolons, quotes, etc) and .eslintrc.json file is setup for you

```
{
  "extends": ["airbnb", "prettier", "plugin:node/recommended"],
  "plugins": ["prettier"],
  "rules": {
    "prettier/prettier": "error",
    "no-unused-vars": "warn",
    "no-console": "off",
    "func-names": "off",
    "no-process-exit": "off",
    "object-shorthand": "off",
    "class-methods-use-this": "off"
  }
}
```
